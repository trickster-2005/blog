Normalisation,正常化,"Organising data into tables so that the results of using the database are always unambiguous and as intended. Normalisation is typically a refinement process after the initial exercise of identifying the data objects that should be in the database, identifying their relationships, and defining the tables required and the columns within each table. First normal form (1NF) is the “basic” level of normalisation: Data and information are contained in two-dimensional tables with rows and columns. Each column corresponds to a sub-object or an attribute of the object represented by the entire table. Each row represents a unique instance of that sub-object or attribute and must be different in some way from any other row (that is, no duplicate rows are possible). All entries in any column must be of the same kind. For example, in the column labelled “Date,” only dates are permitted. In Second normal form (2NF), the tables are in first normal form and, in addition, each column in a table that is not a determiner of the contents of another column must itself be a function of the other columns in the table. At the second normal form, modifications are still possible because a change to one row in a table may affect data that refers to this information from another table. In Third normal form (3NF), the tables are in second normal form and, in addition, there is no transitive functional dependency. For example, if A is functionally dependent on B, and B is functionally dependent on C, then C is transitively dependent on A via B. In Domain/key normal form (DKNF), a key uniquely identifies each row in a table. A domain is the set of permissible values for an attribute. By enforcing key and domain restrictions, the database is assured of being freed from modification anomalies. DKNF is the normalisation level that most designers aim to achieve.",將數據組織到表格中，以便使用資料庫的結果始終明確且符合預期。規範化通常是在初始練習之後的一個細化過程，即識別資料庫中應存在的數據對象、識別它們的關係以及定義所需的表和每個表中的列。第一範式 （1NF） 是歸一化的“基本”級別：數據和資訊包含在帶有行和列的二維錶中。每列對應於整個表所表示的物件的一個子物件或一個屬性。每行表示該子對象或屬性的唯一實例，並且必須在某種程度上與任何其他行不同（即，不可能有重複的行）。任何列中的所有條目都必須是同一類型。例如，在標有“日期”的列中，只允許使用日期。在第二範式 （2NF） 中，表是第一範式，此外，表中不是另一列內容的確定符的每一列本身必須是表中其他列的函數。在第二範式中，仍然可以進行修改，因為對表中的一行的更改可能會影響從另一個表引用此信息的數據。在第三範式 （3NF） 中，表是第二範式，此外，沒有傳遞功能依賴性。例如，如果 A 在功能上依賴於 B，而 B 在功能上依賴於 C，則 C 透過 B 傳遞依賴於 A。在域/鍵範式 （DKNF） 中，鍵唯一標識表中的每一行。域是屬性的允許值集。通過強制執行金鑰和域限制，可以確保資料庫免受修改異常的影響。DKNF 是大多數設計人員希望達到的歸一化水準。,,,,,,,,,,,,,,
